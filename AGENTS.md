# 量化知识库导航

## 这个仓库是什么

一份结构化的量化交易知识库，用 Markdown 文档承载核心概念、策略原理、代码片段和实操笔记，帮助新手系统性地从零搭建自己的量化认知框架。

**目标读者**：有编程基础的量化初学者，希望建立可复用的知识体系而非零散收藏。

**使用方式**：按下方框架逐模块阅读，理解后对照实践，再用自己的话更新或补充文档。

---

## 第二层：数据与因子

> 对应目录：`docs/data/`

### 核心概念

- **K 线数据**：时间周期选择、前复权/后复权、缺失值填充、分钟与日线差异

## 第三层：策略与回测

> 对应目录：`docs/strategies/`

### 策略范式

| 类型 | 原理 | 典型策略 |
|------|------|----------|
| 趋势跟踪 | 跟随已有趋势 | 双均线交叉、海龟交易法、ATR 通道突破 |
| 均值回复 | 价格回归均值 | 布林带上下轨、RSI 超买超卖、配对交易 |
| 统计套利 | 利用统计定价偏差 | 协整配对、跨期套利、ETF 套利 |
| 事件驱动 | 应对特定事件 | 财报公告效应、除权除息套利、IPO 首日 |

### 回测核心原则

- 避免**前视偏差**：回测时只能用已知数据，不可用未来信息
- 注意**幸存者偏差**：使用全样本而非仅存活样本
- 考虑**交易成本**：佣金、滑点、冲击成本
- 做**参数敏感性**分析：避免过度优化（过拟合）

### 回测工具

| 工具 | 适用 |
|------|------|
| Backtrader | 入门友好，策略编写直观 |
| Zipline | 事件驱动回测，学习曲线略陡 |
| 自研向量化回测 | 快速验证因子组合，适合研究阶段 |

## 第四层：实盘与风控

> 对应目录：`docs/trading/`

### 资金管理

- **凯利公式**：最优仓位计算与风险约束
- **固定比例**：每笔交易固定风险比例（如 1% 规则）
- **再平衡**：组合仓位定期调整

### 风控维度

- 单笔止损、日/周最大回撤限制
- 相关性控制（持仓品种不过度集中某行业或资产类）
- 黑天鹅应对：极端行情下的熔断、对冲、减仓规则

### 实盘技术

- 交易接口：CTP（期货）、XTP（股票）、交易所 REST API
- 日志与监控：交易记录、持仓跟踪、异常报警
- 灰度上线：模拟盘 → 小资金实盘 → 加仓

## 实践项目路线

理论学习需要动手巩固。以下是三个由浅入深的实战项目：

### 项目一：双均线策略全流程（第 6 周完成）

```
目标：从数据到回测报告，端到端跑通
输入：50 只 A 股日线数据（AKShare）
输出：一份包含收益曲线、最大回撤、夏普比率的回测报告
知识点：pandas 向量化回测、shift(1) 避免前视偏差、交易成本估算
```

### 项目二：多因子选股模型（第 8 周完成）

```
目标：构建 5 个因子，合成综合得分，分层回测
输入：500 只 A 股日线 + 财务数据
输出：因子 IC 报告 + 分层回测收益曲线 + 多空组合曲线
知识点：因子库、行业中性化、IC/IR 评估、多因子合成
```

### 项目三：实盘模拟盘（第 10 周完成）

```
目标：在模拟盘上跑策略，观察真实市场行为
输入：Binance 市值品种小时线（REST API）
输出：模拟盘交易日志 + 实盘 vs 回测对比
知识点：API 下单、日志记录、异常报警、滑点体验
```

每个项目完成后，将核心代码和结果归档到 `notebooks/` 目录下。

## 文档维护规范

- 每一篇 `.md` 文档都要有前置说明：**内容概述、前置知识、更新时间**
- 代码块中的数据访问路径统一为相对路径，以仓库根为基准
- 文档之间可以互相链接，但不要出现循环依赖
- 每个模块的 `README.md` 作为索引文件，列出该模块下所有文档及其一句话简介
- 参考外部资料（论文、博客）时注明来源链接
- notebooks 目录下的实验完成后，经过验证的内容要写回 docs/ 固化

## 知识下钻规则

学习过程中，读者可能对某个知识点或术语产生追问（如"展开讲讲前复权？"）。这种情况下，执行**知识下钻**：

### 触发条件

- 读者明确要求展开讲解某个概念或术语
- 概念在当前文档中只是一笔带过，不足以支撑理解
- 概念值得独立成篇（有足够的内容深度）

### 术语表结构

所有下钻产生的术语文档统一归入 `docs/glossary/` 目录，按知识模块划分子目录：

```
docs/glossary/
├── README.md             # 术语表总索引
├── basics/               # 基础知识模块
│   ├── forward-adjust.md
│   └── order-types.md
├── data/                 # 数据与因子模块
├── strategies/           # 策略与回测模块
└── trading/              # 实盘与风控模块
```

### 下钻流程

1. **新建术语文档**：在 `docs/glossary/{模块}/` 下创建 `{术语名}.md`，例如 `docs/glossary/basics/forward-adjust.md`
2. **术语文档结构**：
   - 顶部注明父文档来源和前置阅读要求
   - 从更基础的层面重新阐释概念（假设读者第一次接触）
   - 包含代码示例、图表说明、常见误区
   - 末尾给出回到父文档的返回链接
3. **更新父文档**：在对应的知识点处添加指向术语文档的链接标记
4. **更新术语表 README**：在 `docs/glossary/README.md` 中收录新术语的索引
