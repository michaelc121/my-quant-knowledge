# 策略研发方法论

> **前置要求**：已阅读 [strategy-taxonomy.md](strategy-taxonomy.md) 和 [backtest-101.md](backtest-101.md)，理解策略分类和回测原理。
> **内容概览**：策略研发的完整生命周期、从想法到代码的系统方法、实验管理与记录、策略取舍的判断标准。
> **更新日期**：2026-07-02

---

策略研发不是"想到一个策略 → 回测一下 → 可以就上实盘"。它是一套系统化的工程流程，核心目标是把运气和过拟合从决策中剥离出来。

## 一、策略研发的完整生命周期

```
┌─────────────────────────────────────────────────────────┐
│  阶段            产出              决策点               │
├─────────────────────────────────────────────────────────┤
│  1. 想法生成    假设文档          值不值得做             │
│  2. 信号构建    因子/指标代码     信号是否捕捉到了假设   │
│  3. 回测初筛    基础收益曲线     信号有正期望吗         │
│  4. 深度回测    完整绩效报告     扣除成本后还有效吗     │
│  5. 稳健性检验  敏感性/样本外     是不是过拟合          │
│  6. 模拟交易    交易日志         实盘路径跑得通吗       │
│  7. 小资金实盘  实盘绩效         是否与回测一致         │
│  8. 逐步加仓    组合层面表现     是否值得成为核心策略   │
└─────────────────────────────────────────────────────────┘
```

**关键原则：每个阶段的产出必须文档化，失败的原因比成功的原因更重要。**

## 二、阶段 1：想法生成——策略假设文档

所有策略研发的起点是一个**可检验的假设**。在写任何代码之前，先在纸上回答三个问题：

### 假设文档模板

```markdown
# 策略假设：{策略名称}

## 1. 市场现象
描述你观察到的市场行为。
例：A 股中，当 20 日均线上穿 60 日均线后，未来 1-3 个月正收益概率显著高于随机。

## 2. 经济学解释
为什么这个现象存在？如果找不到合理的经济学解释，大概率是数据窥探。
例：20 日均线上穿 60 日均线 = 短期动量超越长期均值 = 趋势确立。
    散户倾向于追涨杀跌，在趋势确立后入场 → 进一步推动趋势。

## 3. 为什么还没被套利掉？
这个现象存在的结构性原因是什么？如果说不清楚，说明你的对手盘比你更聪明。
例：均线交叉信号延迟大（右侧信号），入场时已经错过最优成本。
    超额收益来自承担趋势中断时的回撤风险（风险溢价而非定价错误）。

## 4. 失效条件
在什么市场环境下这个策略会亏钱？预先定义退出条件。
例：持续窄幅震荡（均线反复交叉）→ 频繁开平仓累积交易成本。
    单边急跌（流动性危机）→ 均线滞后导致止损太晚。
```

### 想法筛选的"3 秒法则"

一个策略想法值不值得继续，3 秒钟能回答下面三个"是"：
1. 能用一句话描述清楚？→ 如果描述不清，说明你自己也没想明白
2. 有合理的经济学解释？→ 不能是"因为它历史上赚钱"
3. 你知道谁会亏钱给你？→ 你赚的钱一定有来源

## 三、阶段 2-3：信号构建与回测初筛

### 信号构建的代码结构

```python
# 好习惯：信号生成和回测解耦
# signal.py —— 只负责生成信号，不涉及仓位和资金
# backtest.py —— 接收信号，模拟交易执行
# evaluate.py —— 读取回测结果，生成绩效报告

# 解耦的好处：
# - 同一个信号可以挂不同的仓位管理策略
# - 修改交易成本假设不需要修改信号逻辑
# - 单元测试可以分别写
```

### 回测初筛的最小验收标准

这个阶段的目的是快速淘汰明显无效的策略，不需要完整的绩效报告。

```python
def quick_filter(backtest_result):
    """
    回测初筛：5 个指标决定是否进入深度回测
    任何一项不达标 → 放弃或回炉
    """
    checks = {
        "年化收益 > 无风险利率": backtest_result["annual_return"] > 0.03,
        "夏普比率 > 0.5": backtest_result["sharpe"] > 0.5,
        "最大回撤 < 30%": backtest_result["max_drawdown"] > -0.30,
        "胜率 > 35%": backtest_result["win_rate"] > 0.35,
        "盈亏比 > 1.2": backtest_result["profit_loss_ratio"] > 1.2,
    }
    passed = sum(checks.values())
    total = len(checks)
    print(f"初筛通过: {passed}/{total}")
    for name, result in checks.items():
        print(f"  {'✓' if result else '✗'} {name}")
    return passed >= 4  # 至少通过 4 项
```

## 四、阶段 4-5：深度回测与稳健性检验

初筛通过后，做"试图推翻自己"的深度检验。

### 回测的"刑讯逼供"清单

```python
def stress_test(strategy_func, data):
    """
    深度检验：用各种方式试图证明策略无效
    如果能通过这些测试，策略才算可靠
    """
    results = {}
    
    # 1. 参数敏感性
    for param in strategy_func.parameters:
        for delta in [-0.2, -0.1, +0.1, +0.2]:
            # 参数 ±20% 范围内策略表现不应崩塌
            pass
    
    # 2. 成本敏感性
    for cost_mult in [1.0, 1.5, 2.0]:
        # 成本翻倍后策略是否还有正收益？
        pass
    
    # 3. 时间区段检验
    for period in ["2018-2019", "2020-2021", "2022-2023"]:
        # 每个独立区段策略都赚钱吗？
        pass
    
    # 4. 随机入场检验
    # 将入场信号随机打乱，做 1000 次回测
    # 真实策略的夏普应该 > 95% 的随机策略
    
    # 5. 最大不利偏移
    # 假设每次成交价格都向不利方向偏移 0.5%
    # 策略收益是否仍为正？
    
    return results
```

### 样本外测试——最关键的防线

```python
# 铁律：样本外数据在你做完所有调参之前，一眼都不能看

# 正确分割
train_start = "2015-01-01"
train_end   = "2021-12-31"  # 训练集：调参、选因子、策略迭代
test_start  = "2022-01-01"
test_end    = "2024-06-30"  # 测试集：只跑一次，不做任何修改

# 如果你发现测试集表现不好，回去调整策略，再跑测试集——
# 那测试集已经不是测试集了，它被你"污染"成了第二个训练集。
# 业界做法：测试集的结果只能用于"决定上不上实盘"，不能用于"调参后重跑"。
```

## 五、阶段 6-8：从回测到实盘

### 回测与实盘的预期差异

| 回测结果 | 实盘预期 | 差异原因 |
|---------|---------|---------|
| 年化 20% | 年化 10-15% | 滑点、延迟、市场影响 |
| 夏普 1.5 | 夏普 0.8-1.2 | 成交价差、冲击成本 |
| 最大回撤 15% | 最大回撤 20-25% | 极端行情超出历史范围 |

**经验法则**：实盘表现通常是回测的 50%-70%。如果差异超过这个范围，回去检查回测是否有前视偏差。

### 灰度上线流程

```
模拟盘 1 个月 → 实盘 10% 预算 → 实盘 30% → 全量
        ↓ 任一阶段指标不达标 → 暂停，诊断原因
```

```python
# 每个阶段需要监控的指标
monitoring_metrics = {
    "模拟盘": ["信号执行成功率", "滑点 vs 预期"],
    "小资金实盘": ["收益是否在回测预期范围内", "最大回撤是否超预期"],
    "逐步加仓": ["市场冲击是否显著增加", "相关性是否导致风险集中"],
}
```

## 六、策略取舍的判断框架

### 什么时候继续迭代

- 核心假设没有被推翻，只是参数不够好
- 某个市场环境下表现差，但原因可以解释且可以规避
- 信号方向正确但信噪比低（IR < 0.3）

### 什么时候果断放弃

```python
# 放弃的充分条件（满足任一条即可）
kill_conditions = [
    "样本外 IC 方向反转（正变负或负变正）",
    "核心假设被证伪（如'动量存在于A股'被数据否定）",
    "扣除成本后年化收益 < 无风险利率",
    "最大回撤超过预设上限（如 30%）且无法通过仓位控制降低",
    "策略容量太小——稍微加资金收益就快速衰减",
    "你自己也说不清策略为什么赚钱（说明你发现的是噪音）",
]
```

维护一个 `strategy-graveyard.md`，记录所有被放弃的策略及其死因：

### 实验追踪

每次回测都应该能完整复现。推荐用简单的 `config.yaml` 控制参数：

```yaml
# config.yaml —— 一次回测的完整参数记录
strategy:
  name: "dual_ma"
  version: "v2.3"
  
parameters:
  ma_short: 20
  ma_long: 60
  
data:
  universe: "csi300"
  start_date: "2015-01-01"
  end_date: "2024-06-30"
  adjust: "qfq"  # 前复权

cost:
  commission: 0.0003  # 佣金万三
  slippage: 0.001     # 滑点千一
  stamp_duty: 0.001   # 印花税千一（仅卖出）

risk:
  max_position_pct: 0.95  # 最大仓位 95%
  stop_loss_pct: 0.05     # 单笔止损 5%
```

## 八、研发检查清单

在将策略推进到下一阶段前，逐项检查：

### 回测初筛 → 深度回测
- [ ] 年化收益 > 无风险利率 + 3%
- [ ] 夏普比率 > 0.5
- [ ] 不存在明显的前视偏差（所有信号用 shift(1) 验证）
- [ ] 交易成本已考虑佣金 + 印花税 + 滑点

### 深度回测 → 模拟交易
- [ ] 参数敏感性可接受（±20% 内夏普不崩塌）
- [ ] 样本外测试已运行且结果可接受
- [ ] 最大回撤在预设范围内
- [ ] 策略容量估计已完成

### 模拟交易 → 实盘
- [ ] 模拟盘运行 ≥ 1 个月，信号执行成功率 > 95%
- [ ] 实盘滑点与回测假设一致（差异 < 50%）
- [ ] 日志和告警系统就绪
- [ ] 回撤熔断机制已配置
