# 过拟合

> **来源文档**：[回测原理、常见陷阱与评估指标](../../strategies/backtest-101.md)
> **前置要求**：理解回测的基本概念和评估指标
> **更新日期**：2026-07-02

## 一句话理解

过拟合就是策略记住了历史的"噪音"而不是"规律"，在回测中表现惊艳但实盘一塌糊涂。量化领域最常见的失败模式。

---

## 直观类比

```
考试前你反复刷同一套模拟题，把答案背下来了 → 模拟考满分
换了新题目 → 不及格

策略反复在同一段历史上调参数 → 回测收益 80% 年化
换一段样本外数据 → 亏光
```

---

## 为什么会过拟合

### 1. 参数太多，数据太少

```python
# 危险示例：用 5 年日线数据（~1200 个交易日）调 10 个参数
def overfit_strategy(close, ma_short, ma_long, rsi_period, 
                      rsi_high, rsi_low, atr_period, atr_mult,
                      vol_filter, macd_fast, macd_slow):
    # ... 10 个参数在 1200 个样本上穷举 ...
    pass
```

### 2. 数据窥探（Data Snooping）

在整个数据集上反复测试不同参数组合，最终"发现"了一组恰好拟合历史噪音的参数。正确做法：**调参只能在训练集上做，测试集碰都不能碰。**

### 3. 选择偏差

同时测试 100 个策略，挑出收益最高的那个。即使所有策略都是纯随机信号，也有 ~5% 的策略因为统计偶然性看起来"有效"。

## 如何检测过拟合

### 方法一：样本外测试

```python
# 正确做法：时间线切分
# 2015-2019：训练集（调参、选股、策略优化）
# 2020-2022：验证集（用来确认效果，可以迭代，但注意次数）
# 2023-2024：测试集（只能运行一次，用作最终评估）

train_df = df[df["date"] < "2020-01-01"]
val_df   = df[(df["date"] >= "2020-01-01") & (df["date"] < "2023-01-01")]
test_df  = df[df["date"] >= "2023-01-01"]

# 测试集跑完之前不要再回头看测试集的结果去修改策略
```

### 方法二：参数敏感性分析

一个好的策略应该在参数附近都表现不错，而不是只有一组精确参数才有效。

```python
def param_sensitivity(backtest_func, param_grid):
    """
    参数敏感性分析：看看参数附近的回测表现是否平滑
    
    期望结果：最优参数附近形成一个"高原"而非"尖峰"
    尖峰 = 过拟合特征
    """
    results = {}
    for param_name, param_values in param_grid.items():
        sharpe_list = []
        for val in param_values:
            sharpe = backtest_func(**{param_name: val})
            sharpe_list.append(sharpe)
        results[param_name] = sharpe_list
    return results
```

**判断标准**：调整参数 10%，策略表现不应崩塌超过 20%。

### 方法三：Deflated Sharpe Ratio（DSR）

考虑多次尝试后的夏普比率显著性。简单来说：你试了 100 个策略，夏普 2.0 都不一定显著。

## 预防过拟合的实践清单

| 措施 | 说明 |
|------|------|
| 保持策略逻辑简单 | 能用 3 个参数就不要用 5 个。简单策略的过拟合风险天然更低 |
| 训练/验证/测试分离 | 铁律：测试集只能跑一次 |
| 参数敏感性分析 | 参数附近表现必须平滑 |
| 考虑交易成本 | 过拟合策略往往依赖精确的买卖点，加入真实成本后崩溃 |
| 多市场验证 | 如果在 A 股有效的策略在港股也有效，可信度上升 |
| 记录所有尝试 | 不要只记最好的那次——记录你试了多少次才找到它 |

---

## 常见误区

1. **"样本外表现好就没问题"**：如果你在样本外上迭代了 10 次才得到"好"结果，那样本外已经变成了样本内。真正的样本外只能跑一次。
2. **"参数少就不会过拟合"**：一个参数的策略也可能过拟合——只要你在历史上反复调整那一个参数。
3. **"机器学习天然会过拟合"**：不。XGBoost 有过拟合倾向，但通过早停（early stopping）、正则化和时序交叉验证可以有效控制。
