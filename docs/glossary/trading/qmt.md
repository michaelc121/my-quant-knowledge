# QMT（迅投量化交易平台）

> **来源文档**：[实盘部署与接口选型](../../trading/live-trading.md)

> **更新日期**：2026-07-03

## 一句话理解

QMT 是**迅投**（北京睿智融和，曾用名迅投）开发的一套本地化量化交易终端。它在你的电脑上运行，用 Python 写策略，直接对接券商柜台，实现自动下单。对 A 股散户来说，QMT 是目前门槛最低、功能最完整的实盘量化工具。

## 名字的含义

QMT 全称 "Quantitative Multi-market Trading"（量化多市场交易），但业界直接叫 QMT。它来自**迅投**（Quick Trading Technology），这家公司也做了很多券商的后端交易系统。

## QMT 能做什么

```
QMT 的完整能力：
┌──────────────────────────────┐
│    用户编写的 Python 策略      │
├──────────────────────────────┤
│   QMT 客户端环境               │
│     ├ 行情接收（Level 2）      │
│     ├ 策略执行引擎              │
│     ├ 交易指令发送              │
│     └ 日志/监控                │
├──────────────────────────────┤
│   券商交易网关（券商提供）      │
├──────────────────────────────┤
│   交易所                      │
└──────────────────────────────┘
```

### 核心能力一览

| 功能 | 说明 |
|------|------|
| Python 策略编写 | 内置 Python 环境，用 .py 文件写策略，支持 pandas/numpy |
| 回测 | 内置回测引擎，支持 A 股历史数据回放 |
| 自动交易 | 策略信号直接发到券商柜台下单 |
| [Level 2 行情](../data/level2-data.md) | 十档行情、逐笔成交、委托队列（需券商开通） |
| 多品种 | 股票、ETF、可转债、期货、期权 |
| 本地运行 | 策略在本地执行，延迟低 |

## QMT 的两种形态

### Mini QMT（插件版）

- 以券商 APP 插件的形式运行
- 不需要安装完整的 QMT 客户端
- 直接在券商的交易软件中启动
- 适合小资金、简单策略

### 标准版 QMT

- 独立的桌面客户端
- 功能更完整（多屏监控、策略组合、网格交易等）
- 需要单独安装和登录
- 适合需要复杂策略的用户

两种版本的策略语法基本兼容，但标准版功能更全面。

## QMT 的 A 股适配

QMT 最实用的场景是**已有 A 股账户后想做量化，又不想从头对接券商接口**。你只需：

1. 找一家支持 QMT 的券商开户（银河、国金、中泰、浙商等）
2. 在券商处申请开通 QMT 权限（通常要求资金 1 万-50 万不等）
3. 下载 QMT 客户端或安装 Mini QMT 插件
4. 用 Python 写策略，在 QMT 中运行

一个最简单的 QMT 策略框架：

```python
# 这是一个 QMT 策略的基本骨架
# QMT 的策略以 .py 文件形式存在，放在指定的策略目录下

def init(context):
    """
    策略初始化——QMT 在启动时自动调用
    """
    # 设置要交易的股票
    context.stock = "000001.SZ"  # 平安银行
    context.ma_short = 5
    context.ma_long = 20
    context.position = 0


def handlebar(context):
    """
    每个 K 线周期调用一次——QMT 自动传入当前行情
    """
    # 获取历史收盘价
    close = context.get_close(context.stock, count=context.ma_long)
    
    # 计算均线
    ma_short = close[-context.ma_short:].mean()
    ma_long = close.mean()
    
    # 交易信号
    if ma_short > ma_long and context.position == 0:
        # 买入
        order = context.order_target_percent(
            context.stock, 0.9, "市价单"
        )
        context.position = 1
    elif ma_short < ma_long and context.position == 1:
        # 卖出
        order = context.order_target_percent(
            context.stock, 0, "市价单"
        )
        context.position = 0
```

## QMT 的局限性

1. **Python 版本受限**：QMT 内置的 Python 版本不是最新的，一些较新的第三方库（如 lightgbm、polars）可能装不上或运行异常
2. **回测能力弱**：QMT 的回测引擎不如 Backtrader 或 Zipline 细致，主要用于策略原型验证，深度回测建议导出数据到 Jupyter 中做
3. **数据导出麻烦**：QMT 的行情数据以私有格式存储，导出到外部分析需要写额外脚本
4. **券商绑定**：换券商意味着要重新申请 QMT 权限，策略可能也需要适配新券商的接口差异
5. **策略源码泄露风险**：策略以明文 .py 文件存在本地，如果电脑被人接触则策略完全暴露

## 常见误区

1. **"有 QMT 就不需要自己写代码了"**：QMT 只是运行环境，"策略逻辑"还是要你自己写。它解决的是"怎么写"之后"怎么跑"的问题。
2. **"QMT 的回测结果可以直接上实盘"**：QMT 的回测很粗糙（滑点、冲击成本等都不精确），建议在外部做好深度回测再搬到 QMT 上。
3. **"QMT 只能做股票"**：实际支持多种品种，但不同券商开通的品种权限不同（有的只给股票，有的也给期货/期权）。
4. **"QMT = 量化交易的全部"**：QMT 只是执行端。因子研究、策略开发、回测验证这些核心工作在 QMT 外完成，QMT 只做最后一环：信号执行。
