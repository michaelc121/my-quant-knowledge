# 凯利公式

> **来源文档**：[资金管理与风险控制框架](../../trading/risk-management.md)
> **前置要求**：理解胜率、盈亏比等基本交易概念
> **更新日期**：2026-07-02

## 一句话理解

凯利公式回答"一个信号来了，下注多少才能让长期复利最大化"——下少了浪费机会，下多了有爆仓风险，凯利告诉你最优解。

---

## 公式

```
f = (b × p - (1-p)) / b

其中：
- f：最优下注比例（占总资金的比例）
- b：盈亏比（净收益 / 净亏损，不含本金）
- p：胜率
```

**核心直觉**：
- 盈亏比越大 → 下注越大（赚一次够亏很多次）
- 胜率越高 → 下注越大
- 如果 b × p < (1-p)，即期望为负 → f < 0，不应下注

---

## Python 实现

```python
def kelly_full(win_rate, avg_win, avg_loss):
    """
    完整凯利公式
    
    参数:
    - win_rate: 胜率（0-1）
    - avg_win: 平均盈利金额
    - avg_loss: 平均亏损金额（正数）
    """
    b = avg_win / avg_loss  # 盈亏比
    p = win_rate
    q = 1 - p
    f = (b * p - q) / b
    return max(0, min(f, 1))  # 截断到 [0, 1]

# 例：胜率 40%，平均赚 200，平均亏 100
print(kelly_full(0.4, 200, 100))  # 0.10 → 下注 10% 仓位
```

## 实战中的"半凯利"策略

理论凯利有一个致命问题：它假设你**精确知道**胜率和盈亏比。在实盘中，这两个数字都是估计值。

- 如果你高估了胜率 → 凯利比例偏高 → 实际下注过大 → 亏损加速
- 同时即使凯利比例准确，全凯利的波动也让人难以执行（一天亏 10% 需要极强的心理素质）

**半凯利**（f_kelly / 2）是业界共识：

```python
def half_kelly(win_rate, avg_win, avg_loss):
    return kelly_full(win_rate, avg_win, avg_loss) / 2

# 上例：全凯利 10% → 半凯利 5%
```

半凯利将波动率降低约 25%，长期收益仅下降约 25%，但最大回撤降幅更大，心理可执行性显著提升。

| 70% | 0.5 | 10.0% | 5.0% |

## 凯利的局限性（比公式本身更重要）

### 1. 它假设无限可分

凯利推导中的"每次下注"是独立重复的。在量化交易中，你的信号不是独立事件——今天的走势和昨天高度相关。

```python
# 不要对每笔交易单独用凯利公式
# 应该先用滚动窗口估计当前的胜率和盈亏比，再应用凯利
def rolling_kelly(trade_results, window=50):
    """滚动凯利：基于最近 N 笔交易动态调整"""
    if len(trade_results) < window:
        return 0.02  # 经验默认：2%
    
    recent = trade_results[-window:]
    wins = recent[recent > 0]
    losses = recent[recent < 0]
    
    win_rate = len(wins) / window
    avg_win = wins.mean() if len(wins) > 0 else 0
    avg_loss = abs(losses.mean()) if len(losses) > 0 else 1
    
    if avg_loss == 0:
        return 0.02
    
    return kelly_full(win_rate, avg_win, avg_loss) / 2  # 半凯利
```

### 2. 它忽略最大回撤

凯利优化的是长期增长率，不关心途中你的心情。连续 10 笔亏损（40% 胜率下概率不低），按半凯利也会回撤不少。

**补救**：增加回撤上限约束。如："凯利比例≤总资金的 2%" 或 "如果累计回撤>20%，强制将凯利比例减半"。

### 3. 它不考虑相关性

如果你同时持有 5 个高度相关的头寸，按凯利分别计算然后相加，等于把所有鸡蛋放在一个凯利篮子。

**补救**：按组合相关性折减凯利比例，或直接用组合层面的 VaR 约束替代。

## 从回测指标反算凯利

回测报告中的指标可以直接用来算凯利。不需要额外数据——你的回测结果表里所有信息都已经够了。

### 场景 A：你有逐笔交易记录（CTA / 趋势跟踪）

```python
def kelly_from_backtest_trades(trades_df):
    """
    从回测输出的逐笔交易日志计算凯利
    
    trades_df 至少需要一列: pnl（每笔交易的盈亏金额或比例）
    """
    won = trades_df[trades_df["pnl"] > 0]
    lost = trades_df[trades_df["pnl"] < 0]
    
    if len(won) == 0 or len(lost) == 0:
        return 0.02  # 无可信数据，用默认保守值
    
    win_rate = len(won) / len(trades_df)
    avg_win = won["pnl"].mean()
    avg_loss = abs(lost["pnl"].mean())
    b = avg_win / avg_loss
    
    f = (b * win_rate - (1 - win_rate)) / b
    return max(0, min(f, 1))
```

### 场景 B：你没有逐笔交易记录，但有回测摘要（因子选股 / 指数增强）

用夏普比率推导最优杠杆。大多数回测框架的绩效报告都包含年化收益和年化波动率。

```python
def kelly_from_backtest_summary(annual_return, annual_vol, risk_free=0.03):
    """
    从回测摘要计算凯利最优杠杆倍数
    
    输入可以来自任何回测框架（Backtrader、Zipline、自研引擎）的输出
    """
    excess_ret = annual_return - risk_free
    optimal_leverage = excess_ret / (annual_vol ** 2)
    return optimal_leverage


def kelly_from_sharpe(sharpe_ratio, annual_vol):
    """从夏普比率直接推导，更简洁"""
    return sharpe_ratio / annual_vol
```

### 回测结果反查表

如果你只有回测报告摘要，没有原始数据，可以用下面表格快速估算：

| 回测报告中有 | 可反算的凯利参数 |
|-------------|----------------|
| 总交易数 + 总收益 + 总亏损 | 胜率 = 盈利笔数 / 总笔数；盈亏比 = 总盈利 / 总亏损（近似） |
| 年化收益 + 年化波动率 | 最优杠杆 = (年化收益 - 3%) / (年化波动率²) |
| 夏普比率 + 年化波动率 | 最优杠杆 = 夏普 / 年化波动率 |
| 平均每笔收益率 + 标准差 | 近似为夏普，然后按场景 B 计算 |

### 一个连回测报告都没有时的经验法则

如果完全不知道策略的统计特征，按以下默认值起步：

- **实盘前三个月**：单笔风险 ≤ 0.5%（对应四分之一凯利的保守估计）
- **实盘三个月到一年**：积累交易记录后，用方法一算出真实的凯利
- **实盘一年以上**：数据足够，可以用滚动凯利动态调整

> **核心原则**：宁可低估仓位，也不要高估。低估少赚点，高估可能爆仓。

## 常见误区

1. **"凯利公式就是最佳仓位"**：凯利的前提是你精确知道概率，而这个前提在金融市场中不成立。半凯利更实用。
2. **"凯利公式不用管具体金额"**：凯利比例是资金比例，但对于小资金（<10 万），1% 仓位可能连一手股票都买不到。需要结合实际可交易单位做调整。
3. **"凯利比例是固定的"**：胜率和盈亏比会随市场环境变化。固定凯利 = 在某些时段下注过高或过低。
